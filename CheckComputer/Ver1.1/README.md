# 需求分析，Python重构
外观需求：
1. 若未开启则调起Tool，注意美观，需要隐藏
2. Tool每两小时检查一次
3. 若未激活，则Show NG，半小时未处理，则全屏
4. 若未扫毒，则Show NG，半小时未处理，则全屏


逻辑需求：
1. 检查激活：
	1. 调用Slmgr /dli，检查是否激活
	2. 若正确激活，则显示PASS，否则显示FAIL
	3. 若未激活，则调用Batch档进行激活
	
2. 检查病毒码：
	1. 读取本地的病毒码，和服务器的病毒码
	2. 若相同，则显示PASS，否则显示FAIL
	3. 若程式未打开（同时未尝试打开过），则打开程式，并记录已经尝试打开过
	4. 若版本异常，同时指向有问题（同时未修改过指向），则修改指向，并记录已经修改过指向
	5. 若版本异常，若指向无错（同时未调起过修改GUID的Tool），则运行修改GUID的Tool，已记录已经调用过修改GUID的Tool
	6. 其余异常，人工处理
	
3. 其他需求：
	1. XP和Windows7可以区分处理
	2. 点掉异常对话框
	3. 隐藏托盘图标
